# Code Review

Review code by prioritizing concrete issues over style preferences.

## How It Works

1. Identify behavior changes and affected paths.
2. Look for correctness, security, data-loss, and regression risks.
3. Report findings with file and line references when available.
4. Avoid inventing issues when the code is acceptable.
